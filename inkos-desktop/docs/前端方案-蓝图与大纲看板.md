## 2. 新建书籍向导（Blueprint Wizard）

### 2.1 页面目标
满足文档 0.1/4.1 的约束：N<=5000、不生成标题、先生成阶段规划+前400章 one-liner，系统三态、人物 SOUL core + timeline（后续批次再推导）。

### 2.2 页面结构（Step-by-step）
建议使用 4 步向导（保持节奏与可回退）：
1) 基本信息
   - 书名/题材/logline
   - 预计章节数（最大 5000）
   - 预计总字数
   - 风格标签/视角（可选）
2) 生成设置
   - 初次生成章数：300/400/500（默认 400）
   - 阶段大小：默认 100（可隐藏为高级设置）
3) 生成预览（异步任务）
   - 生成进度条：人物→系统判断→阶段规划→1-100→101-200→…→checkpoint
   - 展示“关键摘要”：
     - 系统三态判断（true/false/optional）+ 简短差异
     - 人物列表（只显示 name/role/SOUL 摘要）
     - 阶段列表（只显示 stage_goal 与 range）
4) 确认发布
   - 发布为 Blueprint v1
   - “进入全书大纲编辑器”

### 2.3 与现有实现的衔接
- 当前已有“创建书籍/项目”的概念（workspace_books + create/switch）：`create_workspace_book`、`switch_workspace_book` 已存在于后端注册列表中（见 [lib.rs](file:///Volumes/MOVE/AI/work/writing/%E8%87%AA%E5%8A%A8%E5%8C%96/inkos-desktop/src-tauri/src/lib.rs#L70-L118)）。
- 本向导完成后：应当自动切换到新建的 book，并跳转到“策划→蓝图”或“策划→大纲”。
- AI 能力与配置读取：沿用 SettingsModal 里的配置（API key/base url/model），并复用现有 llm client 能力。

> 缺口（后端需新增但本次不编码）：蓝图生成与落库命令、阶段规划表、one-liner 表、checkpoint 表、SOUL core/timeline 表。

## 3. 蓝图控制台（Blueprint Console）

### 3.1 页面目标
对“生成阶段规划 + 前400章 one-liner + 校验/修复 + checkpoint”提供可控的运行面板，避免一键跑到底不可控。

### 3.2 关键 UI 区块
- 左侧：当前 Blueprint 版本列表
  - v1（published）、v2（draft）…
  - 支持 Diff/回滚（文字差异，不涉及图形编辑）
- 中间：生成控制台
  - 生成阶段规划（按钮）
  - 生成前 300/400/500（按钮组）
  - 继续生成下一批 +100（按钮）
  - 暂停/继续（用于长任务）
  - 进度：当前批次范围、完成条数、失败原因
- 右侧：检查点与校验
  - 当前 batch checkpoint 预览
  - 校验 issues 列表（error/warn）
  - 一键“按 issues 修复并重新校验”

### 3.3 与现有实现的衔接
- 可复用 WorldBible 的“提取状态提示”交互模式（右上角状态文案）：[WorldBible.tsx](file:///Volumes/MOVE/AI/work/writing/%E8%87%AA%E5%8A%A8%E5%8C%96/inkos-desktop/src/components/WorldBible.tsx#L387-L445)

## 4. 全书大纲编辑器（One-liner Editor，5000章）

### 4.1 页面目标
编辑 5000 行 one-liner，并支持：范围编辑、锁定、范围重生成、版本 diff、范围回滚。

### 4.2 UI 布局建议
- 顶部工具条
  - 跳转：输入章号跳转（1..5000）
  - 范围选择：start/end
  - 操作：锁定/解锁、范围重生成、校验范围、保存版本点
  - 筛选：tag、人物、任务（threads）
- 主体：虚拟表格（强制虚拟滚动）
  - 列：chapter_number（固定）、one_liner（可编辑）、tags、cast_refs、thread_refs、locked
  - 行内编辑：one_liner 文本框（多行但高度限制）
  - 右侧快捷动作：锁定、清空、标记需要重生成
- 侧栏：章节详情（选中行）
  - 当前章关联人物/任务
  - 最近 checkpoint 摘要（便于理解上下文）
  - 校验提示（若该行被 issues 命中）

### 4.3 性能与交互要求（5000上限的关键）
- 列表必须虚拟滚动，仅渲染可视行
- 保存策略：
  - 单行编辑 debounce（例如 500ms）
  - 批量操作一次提交（范围重生成/锁定/标签批改）
- Diff/回滚只做“范围”，不做全书 diff（避免巨量计算）

### 4.4 与现有实现的衔接
短期可先“只做 UI 原型 + 本地 mock”，待后端 one-liner 表与命令补齐后再接入。
中期与写作页联动方式：
- 写作页生成某章正文时，自动从 one-liner 表读取该章 `one_liner + cast_refs + thread_refs + checkpoint` 作为上下文输入。

## 5. 任务看板（Threads Board）

### 5.1 页面目标
主线/支线任务以看板（任务式）管理，无依赖、可并行多条，并可关联章节范围与人物。

### 5.2 UI 结构
- 列（状态）：todo / doing / done / parked
- 卡片字段（最小集）
  - title
  - type（main/sub/character/mystery/growth）
  - goal（可量化）
  - stakes（代价）
  - owner_characters（多选）
  - start_chapter/end_chapter（可空）
  - milestones（checklist）
  - notes
- 与大纲联动
  - 选择卡片后：显示关联章节列表（from one-liner 的 thread_refs）
  - 支持“将选中章节范围批量绑定到某任务”

### 5.3 与现有实现的衔接
现有 Board.tsx 是“统计看板”（hooks/ledger/overview），不是任务式看板，可并存：
- Board：偏诊断/统计
- Threads Board：偏策划/推进管理

若希望合并：可在 Board 里加一个 Tab，切到 Threads Board。
