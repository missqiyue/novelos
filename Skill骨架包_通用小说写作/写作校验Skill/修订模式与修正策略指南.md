# 修订模式与修正策略指南

> 来源：InkOS Reviser Agent
> 用途：根据问题类型选择合适的修订模式，确保修正效果

---

## 一、修订模式概览

### 1.1 五种修订模式

| 模式 | 修改幅度 | 适用场景 | 风险等级 |
|------|----------|----------|----------|
| **polish** | 最小 | 仅表达问题 | 低 |
| **spot-fix** | 局部 | 具体问题点 | 低 |
| **rewrite** | 中等 | 多处问题 | 中 |
| **rework** | 较大 | 结构问题 | 高 |
| **anti-detect** | 全面 | AI痕迹明显 | 中 |

### 1.2 模式选择决策树

```
问题是什么类型？
├─ 表达/用词问题 → polish
├─ 具体段落问题 → spot-fix
├─ 多处分散问题 → rewrite
├─ 场景/结构问题 → rework
└─ AI痕迹明显 → anti-detect
```

---

## 二、polish（润色模式）

### 2.1 适用场景

- 用词不够精准
- 句序需要调整
- 标点节奏问题
- 表达不够流畅

### 2.2 修改规则

**允许**：
- 替换同义词
- 调整句序
- 修改标点
- 删除冗余词

**禁止**：
- 增删段落
- 改变人名/地名/物品名
- 增加新情节
- 改变因果关系
- 改变对话内容

### 2.3 修正示例

```markdown
原文：
他感到很愤怒，因为他知道这是不对的。然而，他不能说什么，因为对方是老板。

润色后：
他的拳头在桌下握紧，指关节发白。他不能说什么——对方是老板。
```

**修改点**：
- 删除"他感到很愤怒" → 用动作暗示
- 删除"因为他知道这是不对的" → 读者自会理解
- 简化转折词"然而" → 用破折号替代

---

## 三、spot-fix（定点修复模式）

### 3.1 适用场景

- 审稿意见指出具体问题
- 问题集中在特定段落
- 其他内容无需修改

### 3.2 修改规则

**允许**：
- 修改问题段落
- 修改问题段落前后各一句

**禁止**：
- 改动无关段落
- 整章重写
- 改变核心事实

### 3.3 修正示例

**问题**：第3段对话太功能化，缺乏性格差异

```markdown
原文：
"你查到了什么？"
"陈志远有财务问题。"
"很好，继续查。"

修正后：
"查到了？"
叶灵点头，却没立刻说话。她看了一眼门口，压低声音。
"晨哥，这事儿……有点大。"
江晨的手指在桌面上敲了两下。
"说。"
"陈志远的账上，有一笔钱流向了……"她顿了顿，"赵氏集团。"
```

**修改范围**：仅修改问题对话段落，前后保持不变

---

## 四、rewrite（改写模式）

### 4.1 适用场景

- 多处分散的问题
- 需要调整叙述力度
- 需要重组问题段落

### 4.2 修改规则

**允许**：
- 重组问题段落
- 调整画面和叙述力度
- 改写问题段落及周边

**禁止**：
- 整章推倒重写（除非问题跨越整章）
- 改变核心事实与人物动机
- 改变剧情走向

### 4.3 修正示例

**问题**：场景太干，缺乏感官细节；对话太功能化

```markdown
原文：
上午十一点，江晨带叶灵去了创业园区。他们走进办公室，江晨坐在椅子上，打开电脑。
"你查到了什么？"
"陈志远有财务问题。"
"很好，继续查。"

改写后：
上午十一点，创业园区的走廊空荡荡的，只有他们的脚步声在回响。
江晨推开办公室的门，一股霉味扑面而来。桌上的灰尘积了厚厚一层，像是很久没人来过。
他拉开椅子，坐下，椅子发出吱呀一声。
"查到了？"
叶灵点头，却没立刻说话。她看了一眼门口，压低声音。
"晨哥，这事儿……有点大。"
```

## 五、rework（重构模式）

### 5.1 适用场景

- 场景组织有问题
- 冲突推进不自然
- 节奏结构有问题

### 5.2 修改规则

**允许**：
- 重构场景推进
- 重新组织冲突
- 调整节奏结构

**禁止**：
- 改变主设定
- 改变大事件结果
- 改变核心人物关系

### 5.3 修正示例

**问题**：场景切换突兀，冲突推进不自然

```markdown
原文结构：
场景1：办公室对话（纯信息传递）
场景2：突然切换到餐厅（无过渡）
场景3：餐厅对话（继续信息传递）

重构后结构：
场景1：办公室对话（有冲突）
  - 江晨追问调查进度
  - 叶灵犹豫，暗示事情复杂
  - 制造悬念：到底查到了什么？
  
过渡：江晨决定换个地方谈

场景2：餐厅对话（冲突升级）
  - 叶灵揭示关键信息
  - 江晨反应：意外+思考
  - 章末钩子：更大的威胁
```

## 六、anti-detect（反检测改写模式）

### 6.1 适用场景

- AI痕迹明显
- 段落等长
- 套话密度高
- 公式化转折多
- 列表式结构

### 6.2 改写手法

| 手法 | 原文 | 改写后 |
|------|------|--------|
| **打破句式规律** | 连续短句 | 长短交替 |
| **口语化替代** | "然而事情并没有那么简单" | "哪有那么便宜的事" |
| **减少"了"字密度** | "他走了过去，拿了杯子" | "他走过去，端起杯子" |
| **转折词降频** | "虽然…但是…" | 用角色内心吐槽或直接动作切换 |
| **情绪外化** | "他感到愤怒" | "他捏碎了茶杯，滚烫的茶水流过指缝" |
| **删掉叙述者结论** | "这一刻他终于明白了力量" | 只写行动，让读者自己感受 |
| **群像反应具体化** | "全场震惊" | "老陈的烟掉在裤子上，烫得他跳起来" |
| **段落长度差异化** | 等长段落 | 有的段只有一句话，有的段七八行 |
| **消灭AI标记词** | "不禁""仿佛""宛如" | 换成具体感官描写 |

### 6.3 修正示例

```markdown
原文（AI痕迹明显）：
他似乎感到了一丝不安。然而，他并没有表现出来。与此同时，他的内心五味杂陈。这一刻，他终于明白了什么才是真正重要的。全场震惊，仿佛时间都静止了。

改写后：
他的手指在桌沿上敲了两下，停住。
老陈的烟掉在裤子上，烫得他跳起来。
"你——"
话没说完，江晨已经站起身，推门走了出去。
```

**修改点**：
- 删除"似乎感到" → 直接写动作
- 删除"然而/与此同时" → 用场景切换
- 删除"五味杂陈/终于明白" → 用行动呈现
- 删除"全场震惊/仿佛" → 用具体人物反应

## 七、修订流程

### 7.1 标准修订流程

```
1. 审计发现问题
   ↓
2. 问题分级（P0/P1/P2）
   ↓
3. 选择修订模式
   ↓
4. 执行修订
   ↓
5. 更新状态文件
   ↓
6. 二次审计
   ↓
7. 通过/继续修订
```

### 7.2 多问题处理优先级

```
P0问题（致命） → rewrite/rework
    ↓
P1问题（严重） → spot-fix
    ↓
P2问题（一般） → polish
    ↓
AI痕迹问题 → anti-detect
```

### 7.3 修订记录模板

```markdown
# 第XXX章修订记录

## 修订模式
- 主模式：spot-fix
- 辅助模式：polish

## 问题清单
1. [P1] 对话太功能化 → 改写对话段落
2. [P2] 转折词重复 → 替换转折词

## 修订内容

### 问题1：对话太功能化
原文：
"你查到了什么？"
"陈志远有财务问题。"

修订后：
"查到了？"
叶灵点头，却没立刻说话。她看了一眼门口，压低声音。
"晨哥，这事儿……有点大。"

### 问题2：转折词重复
原文：然而，他不能说什么。
修订后：他不能说什么——对方是老板。

## 状态更新
- 状态卡：已更新（第X段对话后）
- 伏笔池：无变化
- 角色矩阵：无变化

## 二次审计
- [ ] 问题已修复
- [ ] 无新问题
- [ ] 审计通过
```

## 八、修订禁忌

### 8.1 绝对禁止

| 禁止行为 | 原因 |
|----------|------|
| 改变核心事实 | 破坏连续性 |
| 改变人物动机 | 破坏人设 |
| 改变剧情走向 | 破坏大纲 |
| 改变大事件结果 | 破坏主线 |
| 增加新支线 | 破坏节奏 |

### 8.2 需要同步更新

| 修改类型 | 需要更新 |
|----------|----------|
| 角色行为变化 | 状态卡、角色矩阵 |
| 物品变化 | 资源账本 |
| 伏笔变化 | 伏笔池 |
| 关系变化 | 角色矩阵 |
| 时间变化 | 状态卡 |

### 8.3 修订后检查

- [ ] 核心事实未变
- [ ] 人物动机一致
- [ ] 剧情走向一致
- [ ] 状态文件已更新
- [ ] 无新问题产生

## 九、与现有流程整合

### 9.1 与审计整合

```
审计发现问题 → 问题分级 → 选择模式 → 执行修订
```

### 9.2 与自动修正流程整合

自动修正流程增加模式选择：

```markdown
## 自动修正流程

### Step 1：问题分级
- P0问题：rewrite/rework
- P1问题：spot-fix
- P2问题：polish
- AI痕迹：anti-detect

### Step 2：执行修订
- 按模式规则执行
- 记录修订内容

### Step 3：更新状态
- 更新状态卡
- 更新伏笔池
- 更新角色矩阵

### Step 4：二次审计
- 确认问题已修复
- 确认无新问题
```

### 9.3 与去AI味改写指南整合

anti-detect模式 = 去AI味改写指南的执行版本

- 去AI味改写指南：提供改写方向
- anti-detect模式：提供具体改写手法
