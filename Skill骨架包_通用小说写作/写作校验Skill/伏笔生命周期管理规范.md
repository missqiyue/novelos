# 伏笔生命周期管理规范

> 来源：InkOS伏笔健康管理系统
> 用途：管理伏笔从埋设到回收的完整生命周期，避免伏笔遗忘或烂尾

---

## 一、伏笔生命周期概览

### 1.1 生命周期阶段

```
埋设 → 推进 → 成熟 → 回收 → 闭环
  │      │      │      │      │
  │      │      │      │      └─ 完成闭环，记录结果
  │      │      │      └──────── 回收伏笔，兑现承诺
  │      │      └─────────────── 伏笔成熟，可回收
  │      └────────────────────── 推进伏笔，保持活跃
  └───────────────────────────── 埋设伏笔，建立预期
```

### 1.2 伏笔状态定义

| 状态 | 说明 | 下一步 |
|------|------|--------|
| **dormant** | 休眠中，刚埋设或长期未推进 | 等待推进时机 |
| **progressing** | 推进中，近期有更新 | 继续推进或准备回收 |
| **ready** | 成熟待收，可回收 | 安排回收章节 |
| **resolved** | 已回收，闭环完成 | 记录结果 |
| **abandoned** | 已放弃，不再回收 | 说明原因 |

### 1.3 伏笔类型

| 类型 | 说明 | 典型回收周期 |
|------|------|--------------|
| **immediate** | 立即回收 | 1-3章 |
| **near-term** | 近期回收 | 3-5章 |
| **mid-arc** | 中期回收 | 5-10章 |
| **slow-burn** | 慢热回收 | 10-15章 |
| **endgame** | 终局回收 | 15章以上 |

## 二、伏笔埋设规范

### 2.1 埋设时机

| 时机 | 说明 | 示例 |
|------|------|------|
| **开篇钩子** | 第一章埋设全书主线伏笔 | "三天后，他会来杀你" |
| **场景转折** | 场景切换时埋设小伏笔 | "桌上有一封未拆的信" |
| **对话暗示** | 角色对话中埋设信息差 | "你父亲，还活着" |
| **物品出现** | 关键物品首次出现 | "那把剑上有个奇怪的纹路" |
| **情绪余韵** | 情感高潮后埋设 | "她不知道，这是最后一次见面" |

### 2.2 埋设要求

每个伏笔必须记录：

```markdown
| 字段 | 说明 | 示例 |
|------|------|------|
| **hook_id** | 唯一标识 | H001 |
| **起始章节** | 埋设章节 | 第3章 |
| **类型** | 回收节奏 | mid-arc |
| **表层** | 表面呈现 | 主角捡到一块玉佩 |
| **真实含义** | 实际意义 | 玉佩是皇室信物 |
| **触发条件** | 何时推进 | 遇到皇室成员 |
| **预期回收** | 计划章节 | 第12章 |
| **证据载体** | 物理载体 | 玉佩本身 |
| **状态** | 当前状态 | dormant |
```

### 2.3 埋设密度控制

| 章节 | 新增伏笔 | 说明 |
|------|----------|------|
| **每章** | ≤1个 | 避免伏笔过载 |
| **每批次（3章）** | ≤3个 | 控制伏笔总量 |
| **每卷** | ≤10个活跃伏笔 | 避免读者遗忘 |

## 三、伏笔推进规范

### 3.1 推进方式

| 方式 | 说明 | 示例 |
|------|------|------|
| **信息增量** | 揭示部分信息 | "玉佩上的纹路，与皇室图腾相似" |
| **关联强化** | 与其他元素关联 | "那块玉佩，正是失踪公主的信物" |
| **触发事件** | 触发条件出现 | "皇室成员出现，玉佩开始发热" |
| **角色发现** | 角色注意到伏笔 | "他终于看清了玉佩上的字" |

### 3.2 推进频率

| 伏笔类型 | 最长休眠期 | 推进提醒 |
|----------|------------|----------|
| **immediate** | 1章 | 超过1章未推进需警告 |
| **near-term** | 2章 | 超过2章未推进需警告 |
| **mid-arc** | 4章 | 超过4章未推进需警告 |
| **slow-burn** | 5章 | 超过5章未推进需警告 |
| **endgame** | 6章 | 超过6章未推进需警告 |

### 3.3 推进记录

每次推进必须更新：

```markdown
| 字段 | 更新内容 |
|------|----------|
| **最近推进章节** | 当前章节号 |
| **状态** | dormant → progressing |
| **推进内容** | 本次推进的具体内容 |
| **剩余悬念** | 还有什么未揭示 |
```

## 四、伏笔回收规范

### 4.1 回收时机判断

```
是否满足以下条件？
1. 伏笔状态 = ready 或 progressing
2. 距离上次推进 ≤ 最长休眠期
3. 当前章节适合回收（有相关场景/角色）
4. 回收不会打断当前主线节奏

满足 → 可以回收
不满足 → 继续推进或延迟回收
```

### 4.2 回收方式

| 方式 | 说明 | 示例 |
|------|------|------|
| **直接揭示** | 直接揭示真相 | "原来玉佩是皇室信物" |
| **角色发现** | 角色主动发现 | "他终于知道玉佩的来历" |
| **事件触发** | 事件触发揭示 | "皇室宴会，玉佩被认出" |
| **反转回收** | 意想不到的方式 | "玉佩是假的，真的在别人手中" |

### 4.3 回收要求

回收时必须：

1. **兑现承诺**：回答伏笔埋设时的问题
2. **超出预期**：给读者惊喜，不只是简单揭示
3. **记录结果**：更新伏笔状态为 resolved
4. **清理线索**：确认相关线索已闭环

### 4.4 回收密度控制

| 章节 | 回收伏笔 | 说明 |
|------|----------|------|
| **每章** | ≤1个 | 避免回收过密 |
| **每批次（3章）** | ≤3个 | 控制回收节奏 |
| **每卷末** | 清理活跃伏笔 | 卷末前回收大部分伏笔 |

## 五、伏笔健康检查

### 5.1 健康指标

| 指标 | 健康值 | 警告值 | 危险值 |
|------|--------|--------|--------|
| **活跃伏笔数** | ≤10 | 11-15 | >15 |
| **最长休眠** | ≤5章 | 6-8章 | >8章 |
| **未回收比例** | ≤30% | 30-50% | >50% |
| **推进率** | ≥80% | 60-80% | <60% |

### 5.2 健康检查清单

**每3章检查**：
- [ ] 是否有新伏笔埋设？
- [ ] 是否有伏笔推进？
- [ ] 是否有伏笔达到回收时机？
- [ ] 是否有伏笔超过最长休眠期？

**每10章检查**：
- [ ] 活跃伏笔总数是否健康？
- [ ] 是否有"陈年旧账"需要处理？
- [ ] 伏笔推进率是否达标？
- [ ] 是否需要安排批量回收？

**每卷末检查**：
- [ ] 本卷伏笔是否大部分回收？
- [ ] 是否有跨卷伏笔需要延续？
- [ ] 是否有需要放弃的伏笔？
- [ ] 下卷伏笔计划是否明确？

### 5.3 问题伏笔处理

| 问题 | 处理方式 |
|------|----------|
| **休眠过长** | 安排推进章节，或标记为 endgame |
| **推进困难** | 调整触发条件，或简化回收方式 |
| **回收时机不对** | 延迟回收，继续推进 |
| **无法回收** | 标记为 abandoned，说明原因 |

```markdown
| ... | ... | ... | ... | ... | ... | ... | ... | ... | ... |
```

### 6.2 推进记录

```markdown
## H001 玉佩伏笔

### 埋设（第3章）
- 场景：主角在废墟中捡到一块玉佩
- 表层：普通的玉佩
- 暗示：玉佩上有奇怪的纹路

### 推进1（第6章）
- 场景：主角在古书中看到类似纹路
- 揭示：纹路与古代皇室有关
- 剩余悬念：玉佩的具体用途

### 推进2（第8章）
- 场景：遇到皇室后裔
- 揭示：后裔认出玉佩是皇室信物
- 剩余悬念：玉佩为何出现在废墟

### 回收（第12章）
- 场景：皇室宴会
- 揭示：玉佩是公主的信物，主角与公主有血缘关系
- 反转：玉佩是公主故意留下的
- 状态：resolved
```

## 八、伏笔家族管理

### 8.1 家族定义

**伏笔家族**：同一主题下的多个相关伏笔，共享推进/回收节奏。

示例：
- 家族：主角身世
  - H001：玉佩（皇室信物）
  - H003：神秘信件（父亲消息）
  - H005：胎记（皇室特征）

### 8.2 家族规则

1. **同家族不同时推进**：避免信息过载
2. **家族伏笔集中回收**：同一场景回收多个相关伏笔
3. **家族状态同步**：一个推进，其他也标记活跃

### 8.3 家族台账

```markdown
| 家族ID | 主题 | 包含伏笔 | 整体状态 | 计划回收章节 |
|--------|------|----------|----------|--------------|
| F001 | 主角身世 | H001, H003, H005 | progressing | 第15章 |
| F002 | 父亲失踪 | H002, H006 | dormant | 第20章 |
```
