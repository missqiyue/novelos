# 写作校验Skill包（骨架版）

> 用法：  
> - 每章写完：执行“30秒轻量校验”（本文件）。  
> - 每3章写完：填写《3章回溯审计表》→（如需）走《剧情偏离修正流程》→按《自动修正流程》修一轮→更新《设定圣经》与《连续性索引》→输出“审计与自动修正日志”。

---

## 1) 每章30秒“轻量校验”（写完立刻做）
- [ ] 推进点是否明确（拿到证据/逼退一局/救下一人/签下一纸）  
- [ ] 是否新增/升级一个对抗（对手变招或压迫升级）  
- [ ] 是否有具象细节（物/味/动作链至少1处）  
- [ ] 是否留下新钩子（新问题/新代价/新威胁）  
- [ ] 新名词是否已登记到《连续性索引（名词表）》  
- [ ] 章节标题是否**全书不重名**（对照 `章节标题索引.md`，写前先搜索）  
- [ ] 字数是否合规：1500—2500（正文非空白字符数，不计Markdown标题行）
- [ ] “读者懂得快”：开头200字内是否交代【场景/目标/对抗】三锚？
- [ ] 人设不偏离：本章出场前3位角色的“称呼体系/口吻/做事风格”是否符合 `人物口吻与行为圣经.md`

---

## 2) 每3章硬性要求交付物（缺一不可）
1) `审计与自动修正日志/审计与自动修正日志-第XXX-YYY章.md`  
2) `连续性索引（名词表）.md`
3) `线索管理/人物关系图谱.md`  
4) `线索管理/伏笔_暗线_线索台账.md`
5) `章节标题索引.md`
