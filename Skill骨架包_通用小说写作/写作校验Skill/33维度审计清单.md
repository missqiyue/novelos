# 33维度审计清单

> 来源：InkOS多代理审计系统
> 用途：每章写完后进行全面质量检查，确保内容质量达标

---

## 一、审计流程

### 1.1 审计时机
- **每章审计**：章节完成后立即执行
- **批次审计**：每3章进行深度审计
- **卷末审计**：每卷结束时进行全卷审计

### 1.2 审计优先级

| 优先级 | 类型 | 说明 |
|--------|------|------|
| **P0** | 致命问题 | 必须立即修复，否则不能发布 |
| **P1** | 严重问题 | 建议修复，影响阅读体验 |
| **P2** | 一般问题 | 可选修复，优化内容质量 |
| **P3** | 建议改进 | 锦上添花，不影响主线 |

## 二、33维度详细清单

### A. 内容质量维度（1-10）

#### A.1 核心冲突（dim 1）
- [ ] 本章是否有明确的冲突点？
- [ ] 冲突是否推动剧情发展？
- [ ] 冲突是否有足够张力？
- **评分标准**：0-100分，≥80分通过

#### A.2 角色行为（dim 2）
- [ ] 角色行为是否符合人设？
- [ ] 角色动机是否清晰？
- [ ] 角色决策是否合理？
- **红旗**：角色OOC（Out of Character）

#### A.3 世界一致性（dim 3）
- [ ] 是否违反已建立的世界规则？
- [ ] 设定是否前后矛盾？
- [ ] 新设定是否合理融入？
- **红旗**：世界观崩塌

#### A.4 情节推进（dim 4）
- [ ] 本章是否推进了主线？
- [ ] 是否有明确的推进点？
- [ ] 推进是否自然？
- **红旗**：章节无实质推进

#### A.5 数值/资源一致性（dim 5，可选）
- [ ] 数值变化是否合理？
- [ ] 资源获取/消耗是否记录？
- [ ] 前后数值是否对得上？
- **适用**：有数值系统的题材

#### A.6 对话质量（dim 6）
- [ ] 对话是否有性格差异？
- [ ] 对话是否有信息差/张力？
- [ ] 是否有纯信息传递的对话？
- **红旗**：所有角色说话风格相同

#### A.7 场景描写（dim 7）
- [ ] 是否有五感描写（≥3种）？
- [ ] 场景是否有氛围感？
- [ ] 是否有"干巴巴"的纯叙述？
- **红旗**：场景无感官细节

#### A.8 情感渲染（dim 8）
- [ ] 是否有情感高潮点？
- [ ] 情感是否通过动作/感官呈现？
- [ ] 是否有直白的"他感到XX"？
- **红旗**：情感描写太抽象

#### A.9 节奏控制（dim 9）
- [ ] 开头是否快速入戏（200字内三锚）？
- [ ] 中段是否有起伏？
- [ ] 结尾是否有钩子？
- **红旗**：节奏平淡无波

#### A.10 信息密度（dim 10）
- [ ] 每段是否≤2个新信息点？
- [ ] 规则讲解是否≤2行？
- [ ] 是否有信息过载段落？
- **红旗**：读者消化不良

### B. 结构质量维度（11-19）

#### B.1 开头三锚（dim 11）
- [ ] 场景锚：人在何处？
- [ ] 目标锚：要做什么？
- [ ] 对抗锚：谁来阻止？
- **红旗**：开头200字未交代三锚

#### B.2 章末钩子（dim 12）
- [ ] 是否有新威胁/新秘密/新代价/反转？
- [ ] 钩子是否足够有力？
- [ ] 是否是重复的情绪宣言？
- **红旗**：章末无钩子或钩子太弱

#### B.3 段落结构（dim 13）
- [ ] 段落长度是否有变化？
- [ ] 是否有连续等长段落？
- [ ] 短段落是否用于强调？
- **红旗**：段落等长（AI痕迹）

#### B.4 句式变化（dim 14）
- [ ] 句式是否多样化？
- [ ] 是否有连续相同开头的句子？
- [ ] 长短句是否交替？
- **红旗**：列表式结构（≥3句同开头）

#### B.5 转折词使用（dim 15）
- [ ] 转折词是否重复使用？
- [ ] 同一转折词是否≥3次？
- [ ] 是否可用动作/视角切换替代？
- **红旗**：公式化转折

#### B.6 套话密度（dim 16）
- [ ] "似乎/可能/或许"等是否过多？
- [ ] 套话密度是否>3次/千字？
- [ ] 是否可用确定性叙述替代？
- **红旗**：语气过于模糊犹豫

#### B.7 总结句（dim 17）
- [ ] 是否有"这一刻他明白了"等句式？
- [ ] 是否有解释性旁白？
- [ ] 是否可用动作呈现替代？
- **红旗**：每章都有总结句

#### B.8 连接词密度（dim 18）
- [ ] "然后/接着/于是"是否过多？
- [ ] 是否有流水账特征？
- [ ] 是否可用场景切换替代？
- **红旗**：连接词堆砌

#### B.9 视角一致性（dim 19）
- [ ] 是否有视角跳跃？
- [ ] 信息边界是否准确？
- [ ] 角色是否知道不该知道的事？
- **红旗**：视角混乱

### C. 连续性维度（20-26）

#### C.1 伏笔状态（dim 20）
- [ ] 新伏笔是否已登记？
- [ ] 已有伏笔是否推进？
- [ ] 是否有"陈年旧账"未处理？
- **红旗**：伏笔遗忘

#### C.2 角色状态（dim 21）
- [ ] 角色位置是否连续？
- [ ] 角色状态变化是否记录？
- [ ] 新角色是否已入矩阵？
- **红旗**：角色凭空出现/消失

#### C.3 物品追踪（dim 22）
- [ ] 关键物品去向是否明确？
- [ ] 物品获取/消耗是否记录？
- [ ] 是否有物品凭空出现/消失？
- **红旗**：物品追踪断裂

#### C.4 时间线（dim 23）
- [ ] 时间推进是否合理？
- [ ] 是否有时间矛盾？
- [ ] 季节/天气是否一致？
- **红旗**：时间线混乱

#### C.5 关系变化（dim 24）
- [ ] 角色关系变化是否记录？
- [ ] 关系变化是否有触发事件？
- [ ] 关系矩阵是否更新？
- **红旗**：关系变化无迹可寻

#### C.6 信息边界（dim 25）
- [ ] 角色已知/未知是否准确？
- [ ] 是否有角色"突然知道"？
- [ ] 信息传递是否有迹可循？
- **红旗**：信息边界穿透

#### C.7 支线进度（dim 26）
- [ ] 支线是否有推进？
- [ ] 支线是否服务主线？
- [ ] 是否有支线遗忘？
- **红旗**：支线失控

### D. AI痕迹维度（27-33）

#### D.1 段落等长（dim 27）
- [ ] 段落长度变异系数是否<0.15？
- [ ] 是否有自然的长度变化？
- [ ] 短段落是否用于冲击？
- **检测方法**：计算段落长度标准差

#### D.2 套话密度（dim 28）
- [ ] 套话词密度是否>3次/千字？
- [ ] 是否有"似乎/可能/或许"堆砌？
- [ ] 是否可用确定性叙述替代？
- **套话词表**：似乎、可能、或许、大概、某种程度上

#### D.3 公式化转折（dim 29）
- [ ] 同一转折词是否≥3次？
- [ ] 是否有"然而/不过/与此同时"堆砌？
- [ ] 是否可用自然转折替代？
- **转折词表**：然而、不过、与此同时、另一方面

#### D.4 列表式结构（dim 30）
- [ ] 是否有≥3句连续同开头？
- [ ] 句式是否过于规律？
- [ ] 是否可用不同主语/动作开头？
- **检测方法**：检查连续句子开头

#### D.5 抽象词堆砌（dim 31）
- [ ] 是否有"伟大/宏大/震撼"等词？
- [ ] 是否可用具体画面替代？
- [ ] 是否有"五味杂陈"等抽象表达？
- **抽象词表**：伟大、宏大、震撼、史诗、无与伦比

#### D.6 情绪直白（dim 32）
- [ ] 是否有"他感到很愤怒"等句式？
- [ ] 是否可用动作暗示替代？
- [ ] 是否有"他心中五味杂陈"等？
- **修正方向**：用动作/感官呈现情绪

#### D.7 解释性旁白（dim 33）
- [ ] 是否有"这意味着/他意识到"等？
- [ ] 是否可用剧情呈现替代？
- [ ] 是否有作者跳出来解释？
- **修正方向**：删除解释，用动作呈现

## 三、审计结果处理

### 3.1 问题分级处理

| 分级 | 问题数量 | 处理方式 |
|------|----------|----------|
| **A级** | 0-2个P2问题 | 直接通过 |
| **B级** | 3-5个P2问题或1-2个P1问题 | 轻量修正 |
| **C级** | ≥3个P1问题或1个P0问题 | 深度修正 |
| **D级** | ≥2个P0问题 | 重写 |

### 3.2 修正模式选择

| 模式 | 适用场景 | 修改范围 |
|------|----------|----------|
| **polish** | 仅表达问题 | 只改用词、句序 |
| **spot-fix** | 局部问题 | 只改问题段落 |
| **rewrite** | 多处问题 | 改写问题段落及周边 |
| **rework** | 结构问题 | 重构场景组织 |
| **anti-detect** | AI痕迹明显 | 全面改写降低AI感 |

## 五、审计工具

### 5.1 AI痕迹检测词表

| 类型 | 词表 |
|------|------|
| **套话词** | 似乎、可能、或许、大概、某种程度上、一定程度上 |
| **转折词** | 然而、不过、与此同时、另一方面、尽管如此 |
| **抽象词** | 伟大、宏大、震撼、史诗、无与伦比、五味杂陈 |
| **总结句** | 这一刻他明白了、命运的齿轮开始转动、一切都变了 |
| **情绪直白** | 他感到、他意识到、他明白、他知道 |

### 5.2 段落等长检测

```
计算方法：
1. 统计每段字数
2. 计算平均值
3. 计算标准差
4. 变异系数 = 标准差 / 平均值

判断标准：
- CV < 0.15：段落过于均匀，AI痕迹明显
- CV 0.15-0.30：正常范围
- CV > 0.30：段落变化丰富，节奏感强
```

### 5.3 套话密度检测

```
计算方法：
套话词出现次数 / (总字数 / 1000)

判断标准：
- 密度 > 3：套话过多，需修正
- 密度 1-3：正常范围
- 密度 < 1：套话少，表达直接
```

## 六、与现有流程整合

### 6.1 与写作校验清单整合
- 30秒快速检查 = 写作校验清单A类+B类
- 5分钟标准检查 = 写作校验清单全部
- 15分钟深度检查 = 33维度审计

### 6.2 与自动修正流程整合
- 审计发现问题 → 按问题分级选择修正模式
- P0问题 → rewrite/rework
- P1问题 → spot-fix
- P2问题 → polish

### 6.3 与批次审计整合
- 每3章 → 执行5分钟标准检查 × 3 + 连续性核对
- 每10章 → 执行15分钟深度检查 + 伏笔健康分析
