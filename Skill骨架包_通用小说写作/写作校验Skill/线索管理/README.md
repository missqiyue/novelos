# 线索管理（人物关系图谱 + 伏笔台账）

> 目的：把“关系正确”“伏笔不丢”“暗线可回扣”做成**可查、可更新、可审计**的硬交付物。  
> 适用：长篇连载（≥100章）/多卷结构/需要多线并行的宅斗、权谋、商战、悬疑等。

---

## 1) 本目录交付物（建议每3章更新一次）
1. `人物关系图谱.md`  
   - 用 Mermaid 绘制：阵营/亲缘/婚姻/利益/仇恨/盟约/证据线  
   - 每条关系必须带“**关系阶段**”与“**章节区间**”（例如：误会/合作/并案/成婚）。
2. `伏笔_暗线_线索台账.md`  
   - 以“伏笔ID”管理所有坑、暗线、线索、悬念、钩子  
   - 必填：首次出现章、表层呈现、真实含义、触发条件、计划回扣章、证据载体、状态、回扣去向（本书/续作/外传）。
3. （可选）`回扣排期表.md`  
   - 把“未回扣项”按章节区间排期，避免后期集中爆雷或遗忘。

---

## 2) 更新SOP（强烈建议纳入“每3章硬性要求交付物”）
每完成3章，请按顺序做：
1) 先更新《连续性索引（名词表）》：新增名词、文书/物件去向、证据是否可递官面  
2) 再更新本目录两份文件：  
   - 图谱：新增/变更关系、阶段推进、阵营迁移  
   - 台账：新增≤1条、回收≤1条，更新状态
3) 最后在“审计与自动修正日志”里写一行：  
   - 本批新增/回收了哪些伏笔ID  
   - 仍未回扣的高风险伏笔（≤5条）

---

## 3) 判定标准（什么算“坑已填”）
一个伏笔/暗线，满足任一即算“已回扣”：
- 在正文中**明确落地**（证据到手/人证落名/真相揭示/因果闭环）  
- 或者被明确写成“续作钩子”（在台账中标明去向，并在本书末或卷末留钩）
