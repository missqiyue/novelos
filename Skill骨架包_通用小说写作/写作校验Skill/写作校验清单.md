> 用法：每写完一章，必须逐项检查并打勾。不完成不进入下一章。

## A. 基础校验（5项必做）

### A.1 禁用词检查
- [ ] **破折号"——"**：全文搜索"——"，如有则替换为句号"。"或逗号"，"
- [ ] **分隔符"---"**：正文禁止使用分隔符"---"（包括场景分隔、章节分隔等任何用途）
- [ ] **总结句**：删除"这一刻他明白了/命运的齿轮开始转动/一切都变了"等句式
- [ ] **抽象大词**：替换"伟大、宏大、震撼、史诗、无与伦比"为具体画面
- [ ] **术语禁用扫描**：全章不出现《禁用词与替换词表.md》中的任一禁用词
- [ ] **章节标题不重名**：本章标题未在 `章节标题索引.md` 中出现；并已把本章标题登记入表

### A.2 结构校验
- [ ] **三锚开场**：开头200字内交代【场景/目标/对抗】
- [ ] **章末钩子**：是新问题/新威胁/新代价（非重复情绪宣言）

### A.3 信息密度
- [ ] **规则信息**：每章最多1段，且≤2行
- [ ] **段落减载**：每段≤2个新信息点

### A.4 爽点落地
- [ ] **爽点可验证**：按印/封存/盖章/受理编号/条款号（不要只写情绪胜利）
- [ ] **代价可见**：本章付出了什么代价？（至少一项）

### A.5 人物口吻
- [ ] **主角口吻**：符合人物口吻圣经设定
- [ ] **配角口吻**：不同人物说话风格有差异

---

## B. 吸引力校验（5项必做）

### B.1 流水账检查
- [ ] **无连续3句以上纯叙述**：没有"然后、接着、于是"等连接词过多的段落
- [ ] **每个场景有冲突或情感**：不是单纯"做事"的流水账

### B.2 感官描写
- [ ] **至少3种感官**：视觉/听觉/嗅觉/触觉/味觉（每章至少用3种）
- [ ] **场景有氛围**：不是干巴巴的动作描述

### B.3 情感渲染
- [ ] **有情感高潮点**：本章至少有1个情感高潮（愤怒/恐惧/希望/绝望）
- [ ] **心理描写不直白**：用动作暗示、感官反应，不用"他感到很愤怒"

### B.4 对话质量
- [ ] **对话有性格差异**：不同人物说话方式不同
- [ ] **对话有张力**：有信息差、有潜台词，不是纯信息传递

### B.5 悬念设计
- [ ] **章末有大悬念**：是新威胁/新秘密/新代价/反转
- [ ] **每场景有中悬念**：场景结束时有悬念点
- [ ] **悬念分布自然**：不过度密集，不让读者疲劳

---

## C. 进阶校验（选做≥2项）

### C.1 具象细节
- [ ] 至少1处物（武器手感/枪机回弹/刀口缺口/衣料摩擦）
- [ ] 至少1处味（血腥/臭氧/雨水/汗/火药/药味）
- [ ] 至少1处动作链（冲刺→借力→格挡→反打→脱离）

### C.2 小阻碍
- [ ] 视线被遮/脚下打滑/弹匣卡壳/肩伤牵扯/同伴失位/敌人变招

### C.3 人物小错
- [ ] 按印按歪/封条贴反/名字写错一笔/说话咬舌头

### C.4 对话节奏
- [ ] 短句群（3-5句）+ 1段长句解释（≤2句）
- [ ] 对话允许吞话与反话（不要人人讲道理）

---

## D. 批次校验（每3章必做）

### D.1 情绪节拍
- [ ] 三章内完成【压抑→释放→新压抑】波浪
- [ ] 爽点≥2次/批次

### D.2 主线推进
- [ ] 三章内推进了什么事实？（一句话）
- [ ] 三章内对抗升级点是什么？（一句话）

### D.3 故事链验收
- [ ] 事业链/复仇链：推进了哪条？
- [ ] 情感链/关系链：深化了哪条？
- [ ] 伏笔链：埋下或回收了哪个？

---

## E. 校验不通过怎么办

| 问题类型 | 急救方案 |
|---------|---------|
| 破折号残留 | 全文替换"——"为"。" |
| 缺少爽点 | 加一个可验证的成果（编号/签收/盖章） |
| 钩子太弱 | 章末加一个新威胁/新秘密/新代价 |
| 人物口吻漂移 | 翻阅人物口吻圣经，重写对话 |
| 信息过载 | 拆段，每段只保留≤2个新信息 |

## G. 强制执行规则

1. **每章写完后**：必须完成A类基础校验（5项）+ B类吸引力校验（5项）
2. **每批次（3章）后**：必须完成D类批次校验
3. **校验不通过**：不得进入下一章写作
4. **记录存档**：校验记录保存至`写作校验Skill/校验记录/`目录
