# 通用小说写作 Skill 规范（含写作校验Skill包：设定圣经/大纲/章节生产/审计修正）

> 目标：把"能稳定产出网文长篇"的写作流程固化为可复用 Skill（SOP + 交付物 + 检查清单 + 模板）。  
> 适配：本规范吸收并**完整覆盖** `写作校验Skill` 文件夹内的所有硬性要求与流程（每章轻量校验、每3章回溯审计、偏离修正决策树、去AI味改写、设定圣经更新、连续性索引机制、审计与自动修正日志交付）。
> 
> **InkOS集成增强**：本规范已整合InkOS自动化写作系统的核心功能，包括：
> - 33维度审计系统（见`写作校验Skill/33维度审计清单.md`）
> - 真知文件系统（见`真知文件系统模板.md`）
> - 伏笔生命周期管理（见`写作校验Skill/伏笔生命周期管理规范.md`）
> - 修订模式策略（见`写作校验Skill/修订模式与修正策略指南.md`）
> - 类型配置模板（见`类型配置模板.md`）

## 0. Skill 元信息（建议作为可配置参数）

### 0.1 适配类型（可选）
- 题材：古代/现代/玄幻/悬疑/职场/商战/宅斗/权谋……
- 节奏：爽文/慢热/高密度推理/日常流
- 主线形态：复仇链/升阶链/破案链/商路版图/情感线为辅
- 叙事视角：三/一人称；是否允许短切对手视角（用于节奏错位）

### 0.2 硬性字数规范（必须一致）
- **每章字数：1500—2500（硬性）**
- **默认统计口径**：正文非空白字符数（含标点），不计 Markdown 标题行（`#`/`##`）。
- 若项目改口径（如“纯汉字不含标点”），必须在本规范开头写死，并全书统一。

### 1.2 执行模式：轻量模式 vs 完整模式（双轨制）
> 目的：降低执行成本，避免"流程瘫痪"。实际写作时间应≥60%，填表时间≤40%。

#### 1.2.1 轻量模式（日常批次｜默认使用）
**触发条件**：普通推进批次，无重大转折/高潮/危机

**每章必做**：
- 填写章卡6项（5分钟内）
- 30秒轻量校验（打勾即可）

**每3章必做**：
- 审计与自动修正日志（简化版：只写摘要+红旗+纠偏动作）
- 连续性索引更新（只更新新增名词）

**每10章必做**：
- 人物关系图谱更新（只更新关系变化）
- 伏笔台账更新（只记录新增/回收）

#### 1.2.2 完整模式（关键批次）
**触发条件**（满足任一即启用）：
- 卷首/卷末批次
- 高潮/转折批次
- 主要角色死亡/背叛/黑化批次
- 敌人链升级批次
- 审计发现重大偏离

**每3章必做（完整版）**：
1) **审计与自动修正日志（完整版）**  
   - 路径：`写作校验Skill/审计与自动修正日志/审计与自动修正日志-第XXX-YYY章.md`  
   - 必含：3章摘要、硬点/互证闭环、红旗命中与修正、下一批不可变更约束。
2) **连续性索引更新（完整版）**  
   - 文件：`写作校验Skill/连续性索引（名词表）.md`  
   - 必含：新增名词、别名/称谓、物件去向、文书是否可递官面、证据备份位置。
3) **人物关系图谱（Mermaid）**  
   - 文件：`写作校验Skill/线索管理/人物关系图谱.md`  
   - 必含：阵营/亲缘/婚姻/利益/敌对/同盟；每条关系需写"阶段@章节区间"。  
4) **伏笔/暗线/线索台账（带ID）**  
   - 文件：`写作校验Skill/线索管理/伏笔_暗线_线索台账.md`  
   - 必含：伏笔ID、首次出现章、表层、真实含义、触发条件、计划回扣章、证据载体、状态、回扣去向（本书/续作/外传）。

#### 1.2.3 模式切换规则
```
轻量模式 → 完整模式：
- 连续2批审计发现偏离
- 进入卷末10章
- 发生重大剧情转折

完整模式 → 轻量模式：
- 连续3批审计无偏离
- 进入日常推进期
- 刚完成重大结算后3批
```

## 2. 写作前置：设定圣经（Story Bible）生成规范

> 目的：把“脑内设定”固化为可查的规则与状态，避免长篇写作遗忘/打架。

### 2.1 《小说设定圣经.md》必须包含的 10 大模块
1) **作品定位**：一句话主轴（主线是什么）、爽点承诺、读者预期  
2) **主线三钉（全书不丢）**：  
   - 断局/复仇节点（敌人链条如何被钉死）  
   - 互证闭环升级（猜→可追责）  
   - 版图/权力秩序升级（商路/资格/官面门槛）  
3) **角色卡（关键人物）**：欲望/恐惧/底线/把柄/资源/关系阶段  
4) **势力与机构**：家族/官署/行会/商号/帮会；各自目标与边界  
5) **地理与版图**：城市/街区/码头/关卡/驿站；“货路与卡点”  
6) **规则系统**（必须写边界与代价）：宅规/族规/官规/商规/市场规则  
7) **互证闭环体系**：物证/文书/口供按印/账证；每类证据如何取得、如何封存、如何递交  
8) **术语与称谓**：官称/店铺称/尊卑称谓/行话；统一写法  
9) **伏笔与未解悬念清单**：新增≤3条/批，回收≤3条/批  
10) **连续性索引挂钩**：所有人名/地名/机构/物件/文书必须能在索引里查到

> （新增强烈建议）亲密/暧昧写法请在《小说设定圣经》额外写一节“亲密设定（尺度/密度/多CP策略）”，并与 `战斗与情感写作模板.md` 同步，避免长篇写到中后期尺度漂移、关系失控或亲密戏灌水。

### 2.3（新增）200万字+长篇前置三件套（强烈建议）
在开写正文前，必须先完成这三份文件（否则只会“写到三卷就没油”）：
1) `章节设置与节奏参数.md`（章长/回溯频率/卷赛季结构/每卷结算规则）  
2) `长篇总卷纲_200万字+（模板）.md`（20卷路线图，先粗后细）  
3) `故事链管理台账.md`（五链ID化：主线目标链/敌人链/硬点链/情感链/伏笔链）

### 2.2 “设定圣经更新”的执行频率（硬性）
- **每3章更新一次**（按 `写作校验Skill/设定圣经更新模板.md` 执行）
- 更新时只写“事实与状态”，不要写长篇解释；解释交给正文用动作呈现。

## 3. 大纲规划规范（全书大纲 → 卷纲 → 10章批次纲 → 章卡）

### 3.1 全书大纲（Macro Outline）
至少包含：
- 全书三幕/三段（或四段）结构：开局钩子、中盘升级、终局清算  
- 三钉在全书的“里程碑章节区间”（例如：1-30、31-80、81-完结）  
- 主线与副线的“带宽”：副线必须服务主线（加压/托底/换战法），不得抢主线推进。

### 3.2 卷纲（Volume Outline）
每卷必须写清：
- 卷主题（一句话）  
- 卷小高潮/结算点（≤2条）  
- 卷内“三钉推进点”分别落在哪些章节附近  
- 卷末必须留下“更上层问题”的钩子（进入下一卷的门槛）。

### 3.3 10章批次纲（Batch Outline｜强制与审计绑定）
每一批10章，章纲必须回答：
- 这10章的**事实层推进**是什么？  
- **对抗升级**是什么？（对手变招/压迫升级）  
- **互证闭环升级**是什么？（新增哪一环，存放在哪）  
- **不可逆后果**是什么？（名声/钱/关系/官面风险/人命至少一项）  
- 下一批“不可变更约束”最多5条（用于防跑偏）

### 3.4 单章章卡（Chapter Card｜写之前先写这张卡）
每章必须先定 6 个锚点（写完可用于30秒校验）：
1) 本章推进点（拿到证据/逼退一局/救下一人/签下一纸）  
2) 本章对抗升级（对手变招或压迫升级）  
3) 本章至少1处具象细节（物/味/动作链/小阻碍）  
4) 本章必须出现一个“可验证步骤”（验药/对账/按印/封存/盖到达印）  
5) 章末新钩子（新问题/新代价/新威胁，避免重复情绪宣言）  
6) 新名词登记（人名/地名/机构/物件/文书/称谓）

## 4. 章节生产（写作）工作流（硬性）

## 4.0（新增）“读者懂得快、看的爽”硬规则（写作与润笔共用）
> 目标：让读者**快速抓住矛盾**、**即时获得回报**、**愿意翻页**。  
> 适用于：设计情节、落笔写作、以及每章润笔（自动修正）。

### 4.0.1 快懂三锚（每章开头200字内必须做到）
1) **人在何处**（场景锚）：一句话交代地点/时辰/当下局势  
2) **要做什么**（目标锚）：本章推进点是什么（拿证据/逼退/救人/签纸）  
3) **谁来阻止**（对抗锚）：对手是谁、用什么规则/手段压你

### 4.0.2 爽点三连（每章至少命中其一，最好二连）
- **拿到东西**：证据/契书/口供/回执/编号/钥匙  
- **逼出代价**：让对手背名/具名/写条款号/交出账页  
- **翻转一局**：把对手的规矩反用回去（但必须付代价）

### 4.0.3 信息呈现底线（让读者“懂得快”）
- **先结果后原因**：先把“发生了什么/拿到了什么”写出来，再写怎么做到  
- **每段≤2个新信息点**（人/规则/地点/物件/目的），超了就拆段  
- **规则讲解≤2行**：一句“规矩是什么”，一句“代价是什么”，其余用动作/后果呈现  
- **专有名词首次出现=必须配一个白话解释词**（例：盐引（盐票）、三联回执（三张回单））

### 4.0.4 润笔时的“压缩优先级”（从重到轻）
1) 删解释性总结句（“她意识到/她明白/这意味着”）→换动作  
2) 删重复铺垫（同一情绪/同一威胁讲两遍）  
3) 把长句拆短（读者眼睛要快）  
4) 把抽象词换具象（纸/章/指印/灰/油/刀口）

### 4.1 每章写完立刻做“30秒轻量校验”（只打勾，不复盘）
- [ ] 推进点明确（拿到证据/逼退一局/救下一人/签下一纸）  
- [ ] 本章新增/升级一个对抗（对手变招或压迫升级）  
- [ ] 至少1处具象细节（物/味/动作链/小阻碍）  
- [ ] 章末有新钩子（新问题/新代价/新威胁）  
- [ ] 新名词已登记到连续性索引  
- [ ] 字数合规：1500—2500（按本规范字数口径）
- [ ] 读者“快懂”检查：开头200字内交代了【场景/目标/对抗】三锚（是/否）

### 4.2 每章写作的“表达底线”（去AI味红线）
- 禁止大段解释性旁白与段末总结句（每章至少删1句总结）  
- 抽象词必须配具象支撑（味道/物件/动作）  
- “他意识到/他明白/他知道”改动作链  
- 规矩讲解降噪：每章最多1段规则信息，且≤2行（规矩是什么/谁用它压人/代价是什么）  
- 句式去同质：减少“忽然/仿佛/像/这一刻/不由得”等高频词  
- 白话对白：允许停顿、吞话、反话，不要人人像作者讲课

## 5. 每3章回溯审计 → 偏离修正 → 自动修正（硬性总流程）

> **执行模式选择**：根据1.2节的双轨制规则，选择轻量模式或完整模式。

### 5.0 模式选择（每批开始前判断）

```
是否满足以下任一条件？
- 卷首/卷末批次
- 高潮/转折批次
- 主要角色死亡/背叛/黑化
- 敌人链升级
- 连续2批审计发现偏离

是 → 完整模式
否 → 轻量模式
```

### 5.1 轻量模式流程（日常批次）

**Step 0：生成"3章摘要"（简化版，每层≤80字）**
1) 事实层：发生了什么
2) 红旗检查：有无命中红旗清单

**Step 1：简化审计（只检查核心项）**
- [ ] 推进点是否明确
- [ ] 是否有不可逆后果
- [ ] 字数是否合规

**Step 2：若无偏离，直接进入Step 4**
**Step 3：若有偏离，执行纠偏动作（≤2条）**

**Step 4：更新连续性索引（只更新新增名词）**

**Step 5：输出简化日志**
```md
## 审计日志-第XXX-YYY章（轻量版）
- 摘要：________
- 红旗命中：无/有（____）
- 纠偏动作：________
- 新增名词：________
```

### 5.2 完整模式流程（关键批次）

**Step 0：生成"3章摘要"（三层，每层≤120字）**
1) 事实层：发生了什么  
2) 动机层：关键人物为什么这么做  
3) 规则层：宅规/官规/商规如何起作用，代价是什么

**Step 1：填写《每3章回溯审计表》**
- 模板：`写作校验Skill/3章回溯审计表.md`

**Step 2：若存在偏离（任一核心项≤3分，或命中红旗）**
- 执行 `写作校验Skill/剧情偏离修正流程-决策树.md`
- 产出"纠偏补丁包"：回收伏笔（≤3）/不可逆后果（≥1）/下一批不可变更约束（≤5）/对手变招（≥1）

**Step 3：执行"自动修正流程"（每批必做）**
- 按 `写作校验Skill/自动修正流程.md` 执行（去AI味、规矩降噪、句式去同质、字数补齐或压缩）

**Step 4：更新设定圣经 + 连续性索引（缺一不可）**
- 用 `写作校验Skill/设定圣经更新模板.md` 更新《小说设定圣经.md》对应批次  
- 同步合并进 `写作校验Skill/连续性索引（名词表）.md`

**Step 4.5：更新"线索管理"**
- 更新 `写作校验Skill/线索管理/人物关系图谱.md`：阶段推进/阵营迁移必须与正文一致  
- 更新 `写作校验Skill/线索管理/伏笔_暗线_线索台账.md`：本批新增≤1、回收≤1、未回收≤5（写入审计日志）

**Step 5：输出"审计与自动修正日志"（完整版）**
- 文件名：`审计与自动修正日志-第XXX-YYY章.md`

## 7. “三钉”总规则（避免越写越散｜硬性原则）
1) 每3章必须新增1个不可逆后果（名声/钱/人情/官面风险/人命）  
2) 每3章必须推进1条主线钉子（断局复仇节点 / 互证闭环升级 / 版图+更上层权力秩序）  
3) 规矩不是背景板：能压人也能反用，但反用必须付代价  
4) 人物不说教：信息靠动作、契书、口供、痕迹传递
