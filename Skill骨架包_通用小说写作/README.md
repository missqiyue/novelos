# Skill骨架包（通用小说写作）

> 这是一套完整的网文长篇写作SOP系统，解决三大核心痛点：
> 1. **写长必散** → 通过故事链ID化、赛季结构解决
> 2. **人设崩塌** → 通过人物口吻圣经、3章回溯审计解决
> 3. **伏笔遗失** → 通过伏笔台账、连续性索引解决

**版本信息**：见 `VERSION.md`（当前统一节奏：每3章回溯审计；同书章节标题禁止重名）。

---

## 快速开始

### 方式一：通过Skill调用（推荐）

当用户需要创建新小说项目时，调用"小说写作"Skill：

```
调用Skill：小说写作
```

Skill会自动提供完整的框架和模板。

### 方式二：手动复制骨架包

```bash
cp -r "Skill骨架包_通用小说写作" "/目标路径/新项目名称"
```

## 执行节奏（双轨制）

### 写作标准流程

```
章纲（章纲_卷XX_XX.md）
    ↓
详细章卡（第XXX-XXX章详细章卡.md）
    ↓
章节内容（第XXX章_标题.md）
    ↓
写作校验（写作校验清单.md）← 强制执行
    ↓
下一章
```

> **校验不通过，禁止进入下一章。**

### 轻量模式（日常批次）

**每章必做**：
- 填写章卡6项（5分钟）
- **写作校验清单**（见`写作校验Skill/写作校验清单.md`）
  - 禁用破折号"——"
  - 删除总结句
  - 三锚开场
  - 章末钩子
  - 爽点落地

**每3章必做**：
- 简化审计日志
- 连续性索引更新
- 情绪节拍检查【压抑→释放→新压抑】

**每10章必做**：
- 人物关系图谱更新
- 伏笔台账更新

### 完整模式（关键批次）

**触发条件**：
- 卷首/卷末批次
- 高潮/转折批次
- 主要角色死亡/背叛/黑化
- 敌人链升级
- 连续2批审计发现偏离

**每3章必做**：
1. 完整审计日志
2. 连续性索引更新
3. 人物关系图谱更新
4. 伏笔台账更新

## 核心原则（写死）

1. **三钉不可变**：无论怎么调整，主线三钉不能改
2. **情感线不抢主线**：每3章最多推进1条情感线
3. **代价必须落地**：每个爽点/亲密戏都要有代价
4. **避免降智倒贴**：角色行为要符合人设
5. **五链优先级**：核心链（每3章）> 辅助链（每6章）

## 检查清单

### 每章检查

- [ ] 推进点是否明确
- [ ] 是否有具象细节
- [ ] 是否留下新钩子
- [ ] 字数是否合规（1500-2500）
- [ ] 人设是否偏离

### 每3章检查

- [ ] 是否推进了至少一条故事链
- [ ] 是否有不可逆后果
- [ ] 是否更新了连续性索引
- [ ] 六维检查是否通过

### 每50章检查

- [ ] 三钉是否还在
- [ ] 赛季结构是否清晰
- [ ] 是否有未回收的伏笔
- [ ] 设定圣经是否需要更新
