# 真知文件系统（Truth Files）模板

> 来源：InkOS Truth Files System
> 用途：维护小说的"长期记忆"，确保长篇写作的一致性和连续性

---

## 一、真知文件系统概览

### 1.1 七大真知文件

| 文件 | 用途 | 更新频率 |
|------|------|----------|
| **状态卡** | 当前章节、位置、目标、冲突 | 每章 |
| **资源账本** | 数值/物品/金钱变化 | 每章（有变化时） |
| **伏笔池** | 伏笔埋设、推进、回收 | 每章（有伏笔时） |
| **支线板** | 支线进度和状态 | 每3章 |
| **情感弧线** | 角色情感变化轨迹 | 每3章 |
| **角色矩阵** | 角色信息、关系、已知/未知 | 每3章 |
| **章节摘要** | 每章关键事件和变化 | 每章 |

### 1.2 文件关系

```
设定圣经（静态）
    ↓
真知文件（动态）
    ├── 状态卡 ←── 每章更新
    ├── 资源账本 ←── 有变化时更新
    ├── 伏笔池 ←── 有伏笔时更新
    ├── 支线板 ←── 每3章更新
    ├── 情感弧线 ←── 每3章更新
    ├── 角色矩阵 ←── 每3章更新
    └── 章节摘要 ←── 每章更新
    ↓
写作参考（写作时查阅）
```

### 2.1 模板

```markdown
| 字段 | 值 |

| **当前目标** | ________ |
| **当前限制** | ________ |

| **当前冲突** | ________ |

| 字段 | 值 |
|------|-----|
| **已达成** | ________ |
```

### 2.2 更新规则

**每章必更新**：
- 当前章节
- 当前位置（如有变化）
- 当前目标（如有变化）
- 当前冲突（如有变化）

**有变化时更新**：
- 主角状态
- 当前限制
- 当前敌我
- 已达成/待解决


## 九、真知文件维护流程

### 9.1 每章流程

```
写作前：
1. 查阅状态卡 → 了解当前状态
2. 查阅伏笔池 → 确认待推进/回收的伏笔
3. 查阅角色矩阵 → 确认角色状态和关系

写作后：
1. 更新状态卡
2. 更新资源账本（如有变化）
3. 更新伏笔池（如有伏笔）
4. 添加章节摘要
```

### 9.2 每3章流程

```
1. 更新支线板
2. 更新情感弧线
3. 更新角色矩阵
4. 伏笔健康检查
```

### 9.3 每10章流程

```
1. 全面核对真知文件
2. 检查连续性
3. 检查伏笔健康
4. 整理章节摘要
```

## 十、与现有流程整合

### 10.1 与设定圣经整合

- 设定圣经 = 静态设定（世界观、规则、人物设定）
- 真知文件 = 动态状态（当前状态、变化、进度）

### 10.2 与章卡整合

章卡写作前查阅真知文件：

```markdown
## 第___章《标题》章卡

- 当前状态：________（来自状态卡）
- 待推进伏笔：________（来自伏笔池）
- 角色状态：________（来自角色矩阵）

### 章卡内容
- 推进点：________
- 对抗升级：________
- ...
```

### 10.3 与审计整合

审计时核对真知文件：

- [ ] 状态卡是否更新？
- [ ] 资源变化是否记录？
- [ ] 伏笔是否登记/推进？
- [ ] 角色状态是否一致？
