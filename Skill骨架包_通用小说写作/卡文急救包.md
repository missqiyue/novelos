# 卡文急救包（通用｜直接照做）

> 触发条件：章卡空白超过30分钟 / 写完500字发现是垃圾 / 不知道下一章往哪走。  
> 原理：卡文不是"写不出来"，而是"不知道写什么"或"不知道怎么写"。本包提供**3分钟诊断 → 5分钟动作**的急救流程。

---

## 0) 卡文快速诊断（3分钟）

回答3个问题，找到卡文类型：

| 问题 | 是 | 否 |
|------|----|----|
| 你知道下一章**要发生什么**，但不知道怎么写？ | → **B. 写法卡** | → 下一题 |
| 你有模糊的想法，但**不确定**该往哪个方向走？ | → **A. 剧情卡** | → 下一题 |
| 写出来的东西**读着平淡**，没有情绪？ | → **C. 情绪卡** | → **综合卡** |

---

## 1) A型：剧情卡（不知道往哪走）

### 症状
- 章卡空白，不知道写什么
- 脑子里有很多碎片（场景/人物/设定），但串不起来
- 感觉每个方向都可以，但又都不对

### 急救流程（按顺序做）

**Step 1：回到三钉检查表**
```
这3章批次必须推进哪一颗钉子？
1) 断局/复仇节点推进：____
2) 互证闭环升级：____
3) 版图/权力秩序升级：____

答案：只选1个，写在下面：
________
```

**Step 2：如果三钉都不清楚，用"可验证步骤"倒推**
```
这本章必须出现的一个"可验证步骤"：
□ 验药    □ 对账    □ 按印    □ 封存
□ 盖到达印 □ 复核    □ 追责    □ 交割

选1个，写：________
```

**Step 3：把"可验证步骤"套进一个简单句式**
```
主角做____（动作），结果拿到/逼出/确认____（结果）。
```

**Step 4：如果还不行，用"最小冲突"强行推进**
```
在这章里，对手必须做一件让主角"不得不应对"的事：
□ 公开质疑    □ 卡资源    □ 灭口证人    □ 散布谣言
□ 威胁盟友    □ 篡改数据    □ 提前行动    □ 设陷阱

选1个，写对手的行动：________
```

### 急救动作包（5分钟出稿）

```markdown
## 第___章《___》急救章卡

### 已知
- 不可变更约束：________
- 上一章结尾的钩子：________

### 填空（只填空格，不许想太多）
- 地点：________
- 时间：________（窗口期/副本内/日常/战后）
- 主角现在要：________（动词）
- 有人/有个规则/有个威胁拦他：________
- 他必须先____才能____：________

### 写出章末钩子（先写结局，倒推过程）
章末必须发生：________

### 开写（从章末往前推）
[直接动笔，不要停]
```

## 2) B型：写法卡（不知道怎么写）

### 症状
- 知道要写什么，但写出来像流水账
- 场景干巴巴，没有画面感
- 对话像作者在讲课

### 急救流程（按顺序做）

**Step 1：检查"快懂爽三锚"**
```
开头200字内是否包含：
1) 场景锚（在哪）：____
2) 目标锚（要干啥）：____
3) 对抗锚（谁来拦）：____

补上缺失项：
```

**Step 2：把抽象词换成身体感受**
```
原文：________（抽象词）
改成：________（身体感受）

可用的身体感受替换库：
- 害怕 → 手抖/胃缩/呼吸停/腿软/后退一步
- 愤怒 → 握拳/咬紧牙关/声音变冷/摔东西
- 紧张 → 掌心出汗/喝水呛到/话说一半停
- 犹豫 → 手悬在半空/脚钉在原地/眼神飘移
- 惊讶 → 眼睛睁大/倒吸一口气/后退半步
```

**Step 3：加入一个"可验证步骤"**
```
这章必须出现：
□ 验药 □ 对账 □ 按印 □ 封存 □ 盖到达印 □ 复核 □ 追责 □ 交割

写出动作链：
1) ____（动作）
2) ____（动作）
3) ____（动作）
```

**Step 4：加一个"意外小阻碍"**
```
这章的"小意外"：
□ 东西找不到    □ 门打不开    □ 印泥干了
□ 证人说不认识他 □ 账页少了一页    □ 有人打断
□ 天气坏事（下雨/太热/太冷）    □ 手抖/按印按歪

选1个，用上去：________
```

### 急救动作包（直接照抄）

```markdown
## 写法急救模板（直接套用）

### 五感开写法
他推开____（门/盖子/帘子），____（视觉：看到了什么）。
空气里是____（嗅觉：烧焦/血腥/花香/药味）。
耳边是____（听觉：人声鼎沸/死寂/警报/雨声）。
他感觉到____（触觉：手心汗/金属发烫/布料摩擦）。
嘴里有____（味觉：铁锈/苦涩/血腥/干渴）的味道。

### 打断法（让场景不死）
写到一半，____（人/事/物）突然出现/发生。
____（主角名）不得不先处理：____（另一件事）。
等他回头，____（原来的事）已经____（变化/恶化/消失）。

### 具象细节库（每章至少用1个）
- 物：纸角卷起/墨迹未干/血迹/扣子松了/发簪斜了
- 味：血腥/药苦/尘土/烧焦/花香/铁锈
- 动作链：后退→撞倒→扶住/抬手→停住→收回
- 小阻碍：门闩卡住/灯芯爆花/印泥干裂/账页少一张
```

## 3) C型：情绪卡（写出来平淡无奇）

### 症状
- 写完自己读一遍，完全无感
- 情绪高潮推不上去，读者不会有反应
- 感觉"应该很燃/很虐/很甜"，但读着很平

### 急救流程（按顺序做）

**Step 1：检查情绪类型是否正确**
```
这段要写的情绪是：
□ 爽（翻盘/复仇/拿到东西）
□ 虐（失去/背叛/代价）
□ 甜（信任/靠近/承诺）
□ 燃（战斗/热血/牺牲）

如果不是"一种"，拆成"一种为主"。
这章的情绪主轴是：________
```

**Step 2：如果要"爽"——检查爽点是否落袋**
```
爽点必须有"落袋动作"：
□ 签字/按印 □ 封存 □ 盖戳 □ 编号登记
□ 对方低头/认输 □ 资源到手 □ 证据落袋

这章的爽点落袋是：________
```

**Step 3：如果要"虐"——检查是否落在"选择"上**
```
虐的本质不是"惨"，而是"选择"：
- 两个都想要，只能选一个
- 选了A，失去B
- 以为是A，其实是B的代价

这章的"选择困境"是：________
```

**Step 4：如果要"甜"——检查是否在"克制"后释放**
```
甜不是贴上去就甜，而是：
1) 想靠近（但有阻碍）
2) 终于靠近（一个小动作）
3) 立刻被打断/代价显现

这章的"甜"是：________
```

**Step 5：加一个"情绪炸点"**
```
用以下格式写一个"情绪炸点"：
"（角色名）____（动作），____（结果）。"

示例：
- 他签字的手停了一瞬，但还是按了下去。
- 她想说什么，但手已经先推开了他。
- 他把人拉到身后，自己的手臂挡在了前面。
```

### 急救动作包（情绪爆点库）

```markdown
## 情绪爆点库（每章至少用1个）

### 爽点（必须落袋）
- "他拿起笔，在____上签下名字。"
- "____按在____上，红印盖下去的那一刻，没有退路。"
- "____从____里取出____，放在桌上。'这是你的。'"
- "他当着____的面，把____放进____里。'记录在案。'"

### 虐点（落在选择）
- "她可以____，但代价是____。她选了____。"
- "____知道____，但还是____。因为____。"
- "____想救____，但____只能活一个。他选了____。"

### 甜点（克制后释放）
- "他____（想靠近的动作），停在____（阻碍），最后____（最小的靠近动作）。"
- "她____（想说的话），最后只说了____。（动作）把____递过去。"
- "____的手在____（危险物）前犹豫，最后____（另一个动作）。"

### 燃点（热血+代价）
- "____挡在____前面，'你先走。'"
- "____在____秒内完成____，代价是____。"
- "他____（不可能的任务），只因为____。（代价/原因）"
```

## 4) 综合卡（以上都不对）

### 如果你完全不知道问题在哪
```
停止思考！

立刻做以下动作：
1) 翻到上一章的章末钩子，抄下来
2) 翻到连续性索引，找一个人名/地名/物件
3) 把两者用"____（动作）____（结果）"串起来
4) 直接写

不要停，不要改，写完500字再说。
```

### 如果你陷入"完美主义"
```
告诉自己：
- 烂草稿比空白好
- 现在写的东西90%会被删
- 只有写完才有改的机会

立刻做：
1) 把手机设为飞行模式
2) 把上一章的章末钩子抄下来
3) 写："他____。"
4) 继续写，不要停
```

## 5) 长篇中段卡文专项（300-500章）

### 5.1 症状识别
- 感觉"该写的都写了"
- 不知道下一卷该干什么
- 主角已经很强，没有对手
- 读者反馈"疲了""没新意"

### 5.2 急救流程

**Step 1：检查赛季结构**
```
- 当前在第几赛季？第___赛季
- 本赛季的大结算完成了吗？是/否
- 下一赛季的"更上层问题"是什么？________
```

**Step 2：敌人链升级检查**
```
- 当前敌人链到哪一层？
  [ ] A层个人反派
  [ ] B层规则反派
  [ ] C层系统反派
  [ ] D层终局反派

- 能否让"更高层敌人"露头？
  - 下一层敌人是谁：________
  - 露头方式：________

- 能否让"已钉死的敌人"反扑？
  - 已钉死敌人：________
  - 反扑方式：________
```

**Step 3：版图升级检查**
```
- 主角当前的势力边界是什么？________
- 能否让他"失去"一些东西？
  - 失去什么：________
  - 失去原因：________
- 能否让他"不得不"进入新领域？
  - 新领域：________
  - 进入原因：________
```

**Step 4：情感线升级检查**
```
- 主CP关系到什么阶段？________
- 能否制造一次"大裂缝"？
  - 裂缝原因：________
  - 裂缝代价：________
- 能否让"副CP"承担一次主线功能？
  - 副CP：________
  - 主线功能：________
```

**Step 5：如果都不行**
```
- 回到设定圣经，检查"三钉"是否还在
- 检查是否有"未回收的伏笔"
- 考虑是否该"收尾"或"开新卷"
- 考虑是否该"引入新势力/新规则"
```

### 5.3 中段急救动作包

```markdown
## 中段急救动作包

### 动作A：敌人链升级
- 让更高层敌人露头
- 让已钉死的敌人"借尸还魂"
- 引入"第三方势力"搅局

### 动作B：版图升级
- 让主角"失去"一个据点/资源/盟友
- 让主角"被迫"进入新领域
- 让主角"同时面对"多个战场

### 动作C：情感线升级
- 制造一次"大裂缝"
- 让关系成为"筹码/把柄"
- 让副CP承担主线功能

### 动作D：规则升级
- 引入"新规则"打破旧平衡
- 让规则"反噬"主角
- 让主角"不得不"重新学习

### 动作E：引入新元素
- 引入新势力/新角色
- 引入新副本/新任务
- 引入新规则/新系统
```

### 5.4 中段卡文预防


## 6) 卡文预防清单（写前必做）

每章写之前，回答这5个问题：

1) **本章推进点是什么？**（一句话）
2) **本章对手/威胁是什么？**（必须有）
3) **本章爽点是什么？**（拿到/逼出/翻转/升级）
4) **章末钩子是什么？**（新问题/新代价/新威胁）
5) **有没有一个"可验证步骤"？**（验药/对账/按印/封存/盖戳/复核）

如果5个问题里有任何一个回答不出来，**停，先想清楚再写**。
